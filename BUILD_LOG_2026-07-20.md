# Build Log — 2026-07-20: JSONL Fleet Input for CAGE-1

## Research

Reviewed AGI and agent-architecture work published or updated during the preceding two weeks:

- **Speculate with Memory** (arXiv:2607.12236) uses transition, episodic, and confusion memories for lossless idle-time speculation. The practical signal is that memory may improve prediction without changing the executed trajectory.
- **Self-Aware Recursively Self-Improving Agents for Personal Singularity** (arXiv:2607.12254) proposes signed goal contracts, bounded scopes, validated tools, benchmarks, a fast planner–executor–verifier loop, and a slower evidence-gated improvement loop. It is a systems proposal, not evidence of unrestricted recursive self-improvement.
- **ABot-AgentOS** (arXiv:2607.10350) combines a verifier-aware ReAct-style loop with a typed, source-grounded multimodal graph memory and gated promotion of runtime evolution assets. Its explicit prevention of current-split leakage is relevant to replayable evaluation.
- **Agentic Skill Optimization over Lie Algebroids** (arXiv:2607.11493) treats skill selection and optimization as a structured learning problem. It supports future skill-evaluation experiments but does not justify automatic mutation of this repository.
- **Open-source signals:** `earendil-works/pi` provides a unified LLM API, agent loop, TUI, and coding CLI; `bytedance/deer-flow` is a long-horizon SuperAgent harness with sandboxes, memory, tools, skills, subagents, and a message gateway; `NVIDIA/SkillSpector` scans agent skills for malicious patterns and security risks. These are architecture and security signals, not a controlled popularity ranking.

The strongest common signal is evidence-bounded autonomy: persistent memory, verifiable execution, isolated tools, and reviewable improvement loops matter more than adding an unconstrained self-editing loop.

## Build

Closed the previous carryover with one focused, read-only change:

- `core/cage1_fleet.py` now accepts either a JSON array or newline-delimited JSON (JSONL) snapshots.
- JSONL parsing preserves input order, ignores blank lines, rejects malformed records, and reports the offending line number.
- `cli/cage1_fleet.py` help text now documents JSONL input and output behavior.
- `experiments/test_cage1_fleet.py` adds coverage for ordered JSONL loading and line-specific malformed-record errors.

No policy, memory, retrieval, or self-improvement setting is changed. Malformed records are rejected rather than silently skipped, and the fleet aggregator remains deterministic and non-mutating.
