### Motivation

The CEF substrate integration built on 2026-06-21 (`cef_substrate_integration.py`,
33 tests) and the four prior CEF modules (detector, session, governance helpers,
recorder/breaker bridges) all live behind a single load-bearing bridge helper,
`fold_step_into_cef`. Until today, that helper had **no caller in the loop
substrate** — every AGI-Research agent loop ran without fabrication-aware
verification. The previous run explicitly listed this as the next priority:

> "Wire `fold_step_into_cef` into `AgentLoop.run()` directly: add a
> `cef_session_detectors` dict keyed by `loop_id` to `AgentLoopConfig`; on each
> EVALUATE step, fold the observation's `agent_output` through CEF; if the
> session verdict recommends `freeze`/`restart`/`halt`, surface as an
> ESCALATE verdict to the verifier"
> — `BUILD_LOG_2026-06-21.md`, Next Priority

Today's build closes that gap. Every `AgentLoop` can now opt into the CEF
substrate with a single config field, and the CEF verdict is **load-bearing**:
the session-level signal can PROMOTE the verifier's verdict (CONTINUE →
ESCALATE → HALT) but never DEMOTE it. The conservative posture is preserved.

### Key Components

1. **`AgentLoopConfig.cef_session_detector`** (optional, lazy-import-safe) —
   When set, every act → observe cycle folds through `fold_step_into_cef`.
2. **`AgentLoopConfig.cef_integration_config`** — accepted for API symmetry
   with `create_cef_integration()`; the loop only needs the session detector.
3. **`AgentLoopConfig.agent_output_key`** — operator override for the canonical
   observation key (`agent_output` / `output` / `text` / `response` / ...
   nested `output.text`); defaults to a priority list with a JSON fallback.
4. **`AgentLoop.cef_enabled`** / **`cef_steps`** / **`cef_session_verdict`** /
   **`cef_latest_recommended_action`** / **`cef_audit_path`** /
   **`cef_disabled_reason`** — read-only accessors for the folded CEF data.
5. **`AgentLoop._fold_cef(observation, step_index, turn_index=None)`** —
   per-step bridge that runs the per-output detector, folds into the session
   detector (mutating it in place), appends a `CEFLoopStep` to `_cef_steps`,
   and persists to the sidecar JSONL audit trail.
6. **`AgentLoop._persist_cef(cef_step)`** — writes each folded step to
   `audit_path + ".cef.jsonl"` for replay; only active when CEF is enabled.
7. **`_extract_agent_output(observation, key=None)`** — extracts the agent's
   textual output from an observation dict using a priority list of canonical
   keys with a conservative JSON fallback. The fallback is intentionally
   JSON: it triggers the detector's minimum-length guard on small observations
   and never escalates spuriously on tool traces.
8. **`create_agent_loop(..., cef_session_detector=..., cef_audit_path=...,
   agent_output_key=...)`** — smallest-viable-install convenience wrapper.

### Conservative Posture (the paper's macro invariants)

- CEF override is **additive only**: a clean verifier verdict stays CONTINUE
  when CEF says "continue" or "warn"; an ESCALATE/HALT from CEF upgrades the
  verdict. We never DEMOTE.
- **Per-output CRITICAL (CET) escalates to HALT** regardless of the session
  verdict — the agent faking its own crash is a hard halt signal; the loop
  must not be tricked into continuing.
- **`freeze` / `restart` from the session verdict → ESCALATE**, unless the
  verifier already produced HALT. ESCALATE means: pause and surface; the
  operator can resume by calling `run` again. We do not auto-apply.
- **`warn` → evidence-only**: the `CEFLoopStep` is appended to the EVALUATE
  step's evidence list, and the rationale is annotated with the CEF warning,
  but the verdict is unchanged. The audit trail still records it so the
  operator can grep for the warning.
- **Lazy import**: the CEF substrate is `try`-imported at module load; if
  the substrate is not installed, `cef_session_detector` is silently set to
  None and `cef_disabled_reason` records the import failure. The loop runs
  unchanged.
- **Audit sidecar**: the CEF stream is replay-ready in
  `<audit_path>.cef.jsonl`, independent of the main loop audit. The main
  audit captures the EVALUATE step's composite_rationale (which now
  includes the CEF override prefix) and the appended evidence dicts.

### Conservative Posture — Composite Verdict Promotion Table

| Verifier Verdict | CEF Recommended | Composite Verdict |
|------------------|-----------------|-------------------|
| CONTINUE         | continue        | CONTINUE          |
| CONTINUE         | warn            | CONTINUE (evidence+) |
| CONTINUE         | freeze / restart| ESCALATE          |
| CONTINUE         | halt (CET)      | HALT              |
| ESCALATE         | continue        | ESCALATE          |
| ESCALATE         | warn            | ESCALATE (evidence+) |
| ESCALATE         | freeze / restart| ESCALATE          |
| ESCALATE         | halt (CET)      | HALT              |
| HALT             | anything        | HALT              |

The composite is **always ≥ max(verifier, CEF)**, never less.

### Research Synthesis

- The CEF substrate is now end-to-end wired into the loop. A single
  `AgentLoopConfig(cef_session_detector=...)` flag activates all four prior
  CEF modules (detector, session, governance, recorder/breaker) on every
  iteration.
- The conservative-promotion invariant is implemented as a 9-cell table
  rather than a side-effecting upgrade function. The table is the load-bearing
  claim; the table's additivity is the safety property.
- The audit trail now carries **two parallel streams**: the main loop's
  EVALUATE steps (which include the composite rationale and the CEF evidence)
  and the CEF sidecar (every folded step's full data including
  `output_hash` and `detection_id`). Operators can replay the CEF history
  from either stream.
- The per-step `CEFLoopStep.recommended_loop_action` is the bridge's
  contract; the loop's verdict is derived deterministically from the
  (verifier, CEF) pair. This makes the bridge testable in isolation —
  every test in this file exercises the bridge without touching the
  underlying detector's pattern catalogue.
- The `_extract_agent_output` helper's fallback to JSON-serialized
  observation is the conservative analog of the `BrakePedal`'s "BRAKED
  hides all proposals" posture: when the operator's work function doesn't
  surface a clear agent output, the loop records the raw observation but
  doesn't let it dominate the CEF signal.

### Why this architecture is right (for the next run)

- The four-layer safety substrate is now **first-class** in the loop:
  Signal (PIGEngine) → Action (SafetyCircuitBreaker, now bridged) → Routing
  (ThreeRingGovernor) → Output (CEF detector + session detector) →
  Causal (EvidenceLedger, now bridged) → Loop (AgentLoop, now bridged).
- The bridge's load-bearing claim is **CEF never demotes**. This is the
  substrate-level statement of the conservative posture; every other layer
  makes the same claim in its own terms.
- The lazy import preserves the minimal-install path: a loop that does not
  import CEF substrate still works, and `cef_enabled` returns False.
- The sidecar audit is independent of the main loop audit, so the CEF
  stream is replay-ready even if the main audit is truncated.
