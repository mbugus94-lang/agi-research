## Build Task: CEF Session-Band Integration (`assess_cef_to_breaker`)

**Motivation**

The 2026-06-28 build's documented next-priority was:

> `assess_cef_to_breaker` session-band integration: when `assess_cef_to_breaker` is called and the session is in `POINT_OF_NO_RETURN`, surface the *session* band (`trip_band` from the session verdict) in the breaker outcome alongside the *per-output* band. The operator sees both layers' measurements in one place — and now both layers feed the same `GovernorCircuit`.

The previous outcome carried only the **per-output** band (measured, from the `ProbabilisticTripEngine`) and the **session state** as a categorical enum. The session layer's *band* — the operator's vocabulary for "how bad is the session" — was implicit. Today closes that gap: the outcome now exposes `session_band` (categorical, derived from `CEFSessionState`) alongside `trip_band` (measured, derived from the engine's upper bound).

**What was built**

`core/cef_substrate_integration.py` — additive extensions only:

1. **`band_for_session_state(state)`** — categorical analogue of `band_for_upper_bound`:
   - `CLEAN` → `"low"`
   - `EARLY` → `"low"`
   - `WARNING` → `"medium"`
   - `POINT_OF_NO_RETURN` → `"critical"`
   - `None` / unknown → `"unknown"`
   Mirrors `CEFSessionAction` recommendations in spirit: CONTINUE→LOW, WARN→MEDIUM, FREEZE/RESTART→CRITICAL.

2. **`session_should_trip(session_verdict, trip_on_horizon, trip_on_restart)`** — pure helper that returns `True` iff the session layer *alone* would trip the breaker. Mirrors the session-level branch in `assess_cef_to_breaker` exactly, deterministic and easy to test.

3. **`CEFBreakerOutcome` two new fields** (both default-typed so existing call sites are unaffected):
   - `session_band: str = "unknown"` — categorical session-layer band
   - `session_should_trip: bool = False` — session-layer trip recommendation

4. **`CEFBreakerOutcome.to_dict()`** extended to expose both new fields.

5. **`assess_cef_to_breaker` population** — both fields populated whenever a session verdict is supplied; rationale message extended with `session_band=...` and `session_should_trip=...` so callers can see both layers in one string.

**Research Synthesis**

The 2026 substrate-vocabulary convergence (`"stateless LLM + stateful runtime"`) settles into a *three-band* reporting pattern: **per-output band (measured)** — from the engine's `(1-α)` upper bound, **session band (categorical)** — from `CEFSessionState`, and **action verdict (boolean)** — from the OR of the two layers' trip recommendations. The 2026-06-26 run merged the *policy* layer (`GovernorCircuit`); today's run merges the *observation* vocabulary so the breaker outcome carries both layers' measurements in one place — exactly what the substrate needs to feed `GovernorCircuit` more informatively on future runs.

**Why this matters**

- The per-output band answers "how bad is *this* output?"
- The session band answers "how bad is *this session*?"
- The trip verdict answers "should the agent's principal be blocked?"
- All three are now reported in one `CEFBreakerOutcome`, in the same vocabulary, with the same rationale string.

The next caller that wants to gate on band-rather-than-trip (e.g., "trip only when both bands are MEDIUM+") can do so off `outcome.session_band` and `outcome.trip_band` without re-implementing the categorical mapping.
