### Build Task: Stateful policy gate driven by `ProbabilisticTripEngine` bounds

**Motivation**: every safety measurement we've built so far — per-output `CEFBreakerOutcome.trip_upper_bound`, session `CEFSessionVerdict.trip_upper_bound` — produces a sound (1 - alpha) bound on the agent's failure rate. But a measurement that nobody reads is a hidden alarm. Today's build is the operational instrument: a stateful policy gate that *acts* on the bound. The agent's substrate gains a third leg of safety — measurement (`ProbabilisticTripEngine`), per-session observation (`CEFSessionDetector`), and now *policy* (`GovernorCircuit`). This closes the 2026-06-25 next-priority item.

### Key Components

1. **`core/governor_circuit.py` (~1100 lines, new)** — `GovernorCircuit` + `GovernorCircuitConfig` + `GateState` (CLOSED / OPEN / HALF_OPEN) + `GateAction` (ALLOW / BLOCK / PROBE) + `GateObservation` + `GateTransition` + `GovernorDecision`. Mirrors the `SafetyCircuitBreaker` contract already in the repo (3-state gate, append-only audit, decision dataclass).
2. **Three-state hysteresis** with the invariant `reset_threshold < trip_threshold` (enforced in `__post_init__`):
   - CLOSED -> OPEN when `bound >= trip_threshold` (with warm-up grace; see below)
   - OPEN -> HALF_OPEN when `bound <= reset_threshold`
   - HALF_OPEN -> CLOSED when `consecutive_good >= probe_count`
   - HALF_OPEN -> OPEN on rebound (recovery-failure resets `consecutive_good`)
3. **Two-layer max** when two engines are configured (per_output + session): the gate's bound is the *maximum* of the two engines' bounds. The gate trips when EITHER layer reports a high bound; recovers when BOTH are low. The trip band is the union of the per-output and session bands. Mirrors the paper's "max aggregation across independent estimators" convention.
4. **Cold-engine-silent posture** (conservative-by-default fix for the integration-test bug):
   - A fresh `ProbabilisticTripEngine` with `Beta(1, 1)` prior and zero observations has `trip_upper_bound = 1.0` (CRITICAL) because the prior is uniform on [0, 1]. Without treatment, the gate would trip instantly on the first feed — the opposite of conservative behavior.
   - The gate now checks `engine.n_observations`. If `n_observations == 0` or the engine is `None`, the engine is **silent** (contributes bound=0 to the two-layer max). A cold engine means "no evidence", not "maximal risk".
   - This matches the AgentRiskBOM (arXiv:2606.21877) distinction between "capability opacity" and "demonstrated behavior": absence of evidence is reported alongside evidence, never conflated.
5. **Warm-up grace window** (the policy refinement on cold-engine-silent): `config.warm_up_min_obs` (default 5) — during the first N observations, the gate reports bounds and band decisions normally but suppresses the CLOSED -> OPEN transition. The suppressed decision is recorded as a `warming_grace` transition so the audit trail is complete. This prevents the gate from tripping on a single first observation's wide posterior quantile (Beta(1+1, 1) bound ~ 0.78 for one no-trip observation).
6. **`n_observations` property on `ProbabilisticTripEngine`** (10 lines added to `core/cef_probabilistic_verification.py`) — alias for `n_total` so the `BoundSource` protocol can express cold-vs-warmed state explicitly. Backward-compatible.
7. **Append-only transition log** with `write_audit(path)`: every state change (including suppressed warming_grace changes) is recorded with timestamp, bound, band, reason, and consecutive_good counter. JSONL output is idempotent on repeated calls.
8. **`decide()` is read-only** — does not transition state, does not increment `consecutive_good`. Only `feed(observation)` updates state. `decide()` is the "what would the gate do *now*" query for callers that want to gate an action without recording an observation.

### Conservative posture (the paper's invariants)

- Trip band is the union of the per-output and session bands.
- Transitions are append-only; the transition log is the audit trail.
- The bound is never demoted by the gate (only by the engines themselves).
- `reset_threshold < trip_threshold` is enforced in `__post_init__`.
- The OPEN -> HALF_OPEN transition itself counts as the first good probe.
- `write_audit` is append-mode and idempotent on repeated calls.
- HALF_OPEN -> OPEN recovery-failure resets `consecutive_good` to 0.
- Cold engines are silent — bound=0 contribution to the two-layer max.
- Warm-up grace is conservative: a gate that might otherwise have tripped stays CLOSED, with the suppressed decision recorded in the audit log.
- Engines that do not expose `n_observations` (legacy stubs) are assumed warmed — back-compat with prior tests.

### Bug found and fixed during this build

The original integration tests (`TestGovernorCircuitIntegration`) used default `Beta(1, 1)` priors and `trip_threshold=0.20`, expecting the gate to be CLOSED after 20 ok observations. The math actually worked: Hoeffding+Clopper-Pearson bound at n=20, 0 trips is 0.1329, which is below 0.20. The real bug was the first-feed trip: a fresh engine had bound=1.0 (CRITICAL) and the gate would trip on the very first observation. Fixed with cold-engine-silent + warm-up grace. The integration tests now pre-warm with 20 ok observations before driving trips, and use looser `trip_threshold=0.40` / `reset_threshold=0.20` for the integration scenarios (the conservative defaults 0.05/0.01 are designed for production with calibrated priors, not for unit tests with the default uninformative prior).

### Research Synthesis

The 2026 consensus ("stateless LLM + stateful runtime", genalphai.com; Springer agentic AI review) is that the stateful runtime is where the *measurable* safety lives. Our substrate now has three layers on that axis: (1) `ProbabilisticTripEngine` is the *measurement* (sound (1-alpha) bound on trip rate), (2) `CEFSessionDetector` + `CEFIntegrationConfig` is the *observation* (per-output and per-session wiring), and (3) `GovernorCircuit` is the *policy* (stateful gate that acts on the bound).

**AgentRiskBOM (arXiv:2606.21877, accepted IEEE Cyber-AI 2026, Jun 25 2026)** independently arrived at the same three-layer decomposition: capability inventory (our `ThreeRingGovernor.ring_layer` + `AgentDescriptor.capabilities`) + control lineage (our `GovernorCircuit` transitions + `ProofCarryingAction` certificates) + evidence ledger (our `EvidenceLedger` + `GovernorCircuit.transitions()` JSONL). Different vocabulary, identical architecture.

The Hephaestus (v0.7.23), Hermes Agent v0.15.0, Continuum, and OpenLAIR/OpenSkill projects are all converging on the same pattern: a thin verifier layer above a stateful memory layer. `GovernorCircuit` + `ProbabilisticTripEngine` + `CEFSessionDetector` is the closest functional equivalent in our substrate, with the bound as the verification gate and the JSONL transition log as the audit trail.

The cold-engine-silent posture is the operational form of "absence of evidence is not evidence of absence" — but only when that absence is itself a fact about the engine's *state*. The gate distinguishes "no engine" (silent, bound=0), "cold engine" (silent, bound=0), "warming engine" (visible bound but trip suppressed), and "warmed engine" (full policy input). This four-state distinction is what makes the gate conservative-by-default: the operator's `trip_threshold` only applies when there is *evidence* to evaluate.
