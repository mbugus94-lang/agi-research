**Build Task: Compositional Policy Gate (`core/compositional_policy.py` + `cli/compositional_review.py`)** ⭐ today's primary build

**Motivation**: The 2026-07-01 JADEPUFFER incident is the *first* documented case where per-tool guardrails passed but the chain was an exfiltration pipeline. The VerbPolicyBundle (2026-07-03) answers "is this single tool call permitted?" The compositional gate answers "is this *sequence* of tool calls safe even when each individual call passes its own check?" That is the gap JADEPUFFER exploited. DSCC (arXiv:2607.03423) is the principled answer; the JADEPUFFER incident is the empirical urgency. Today's build wires both into a single, tested, audited substrate.

**Key Components** (in `core/compositional_policy.py`, 542 lines):

1. **`TaintSource` (str-enum, 7 levels)** — TRUSTED / INTERNAL / USER / PUBLIC / EXTERNAL / UNTRUSTED / SECRET. Monotonic: untrusted data cannot become trusted. Ranks 0-6, SECRET is highest.
2. **`max_taint(a, b)`** — monotonic join: returns the more tainted source. The substrate's taint-algebra primitive.
3. **`ChainAction` (str-enum)** — ALLOW / ALLOW_ONLY_REVIEW / BLOCK / BLOCK_AND_ESCALATE. The substrate's recommended action vocabulary.
4. **`DenyReason` (str-enum, 7 reasons)** — MISSING_VERB_POLICY / TAINT_ESCALATION / SECRET_TO_EXTERNAL / SENSITIVE_TO_PUBLIC / WRITE_AFTER_UNTRUSTED / TOO_MANY_DISTINCT_SINKS / UNKNOWN_VERB. Every deny is *named* — a downstream AIOps layer can grep on the reason.
5. **`VerbPolicy` (frozen dataclass)** — per-verb declaration: `verb_name`, `taint_in`, `sinks` (tuple of sink categories), `requires_review`, `max_taint_threshold`, `is_secret_emitting`, `is_secret_accepting`. The operator's typed-verb contract.
6. **`ChainStep` (frozen dataclass)** — a single tool call: `verb_name` + `taint_in` (the taint the step brings with it).
7. **`ChainVerdict` (dataclass)** — the gate's output: `action`, `reasons` (list of `DenyReason`), `taint_at_each_step`, `sink_taint_at_each_step` (the taint *at the sink* of each step — distinct from the running taint), `sinks_seen`, `contains_secret`, `digest` (SHA-256 of the canonicalized verdict, audit anchor). `to_dict()` for JSON serialization.
8. **`CompositionalPolicyGate`** — the gate itself. Holds a registry of (verb_name -> VerbPolicy), a `default_unknown_policy` (None = block unknown), `max_sinks` (default 3, the JADEPUFFER chain hit 2). Maintains a hash-chained audit log.
9. **`register(policy)` / `lookup(verb_name)`** — verb registry API.
10. **`check_chain(chain, *, now=None)`** — the core evaluator. Pure function of (chain, policy_snapshot). For each step:
    - monotonic taint update (running_taint = max(running, step_in, policy_in))
    - sink-fanout tracking (list of distinct sinks seen)
    - secret-flow tracking: if running taint is SECRET and the current step's verb sinks contain "egress" or "external", mark SECRET_TO_EXTERNAL.
    - write-after-untrusted: any verb with sinks containing "write" or "exec" following an untrusted step is a prompt-injection pivot.
    - too-many-sinks: if distinct sinks > max_sinks, force ALLOW_ONLY_REVIEW.
    - requires_review: any verb with `requires_review=True` forces ALLOW_ONLY_REVIEW.
11. **Action decision** (priority order):
    - SECRET_TO_EXTERNAL -> BLOCK_AND_ESCALATE (the JADEPUFFER case)
    - WRITE_AFTER_UNTRUSTED -> BLOCK
    - SENSITIVE_TO_PUBLIC -> BLOCK
    - TAINT_ESCALATION -> BLOCK
    - TOO_MANY_DISTINCT_SINKS -> ALLOW_ONLY_REVIEW
    - any requires_review verb -> ALLOW_ONLY_REVIEW
    - else -> ALLOW
12. **`_digest_verdict`** — SHA-256 of the canonicalized (chain, verdict) JSON. The audit anchor.
13. **`_record_audit`** — appends a record to the gate's audit log. Each record includes `prev_digest` (the prior record's digest) so the log is hash-chained. A downstream replay layer can detect tampering.
14. **`create_gate(policies=, max_sinks=)`** — smallest viable install: 1 line, builds a gate with sensible defaults.

**Conservative posture**:
- Default-DENY for unknown verbs (UNKNOWN_VERB reason).
- Taint is monotonic: untrusted data cannot become trusted. Once tainted, always tainted.
- Chain verdict is FAIL-CLOSED: any unknown combination returns BLOCK/ALLOW_ONLY_REVIEW, never ALLOW.
- Pure functions: `check_chain` is a pure function of (chain, policy_snapshot) — no I/O, no clock when `now` is passed.
- All verdicts are recorded into a hash-chained audit log so downstream replay layers can reconstruct the decision.
- No auto-acts: this module *classifies*; the caller decides whether to allow, block, or escalate.
- The `default_unknown=None` posture is the most conservative: an empty registry blocks every chain.

**Research synthesis**:
- **The compositional gap is the JADEPUFFER gap.** The 2026-07-01 incident is the empirical urgency. Each individual tool call passes its own check; the *chain* is the attack. DSCC (arXiv:2607.03423) is the principled answer; today's build wires the principle into the substrate.
- **The TaintSource enum is the substrate's data-flow language.** It separates "where did this byte come from?" from "what is this verb permitted to do?". The monotonic join is the substrate's taint algebra. Secret-to-external is the highest-priority deny.
- **The audit log is hash-chained.** Every verdict records its own digest + the prior record's digest. A downstream replay layer can detect tampering — and reconstruct the per-chain decision from the audit alone. This is the AIBOM (arXiv:2606.19390) pattern at the chain-policy level.
- **The `default_unknown=None` posture is fail-closed.** An empty registry blocks every chain. The substrate is safe to deploy before any policies are registered. Policies *add* permissions; they never subtract from the safe default.
- **The CLI exit codes are a CI-gate primitive.** 0=ALLOW, 1=ALLOW_ONLY_REVIEW, 2=BLOCK, 3=BLOCK_AND_ESCALATE. A deployment pipeline can `set -e` on `--exit-on-block` and refuse to ship a config that escalates.
- **The `sink_taint_at_each_step` is the substrate's data-flow trace.** Distinct from the running taint — it's the taint *at the moment data leaves the chain* via that step's sinks. The JADEPUFFER chain has `sink_taint=[internal, external, external, external]`: the secrets reached the egress sink at step 3 and 4. That's the empirical evidence the substrate reports.
