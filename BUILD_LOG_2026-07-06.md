**Key components**:

1. **`AdvisoryCategory` (str-enum)** — `GENERIC / CEF_EMERGENCE / POLICY_LOOSENING / THANATOSIS / BUNDLE_GAP`. Priority: THANATOSIS > POLICY_LOOSENING > CEF_EMERGENCE > GENERIC. The category is computed from component properties (no operator input).
2. **`AdvisorySeverity` (str-enum)** — `NONE / LOW / MEDIUM / HIGH / CRITICAL`. Mirrors CEFSeverity ordering; `rank()` classmethod for ordinal comparison.
3. **`AdvisoryAction` (str-enum)** — `DEFER / MONITOR / REMEDIATE / BLOCK`. Operator-facing recommendation. Conservative ladder: CRITICAL+thanatosis → BLOCK, HIGH+ → REMEDIATE, MEDIUM → MONITOR, LOW/NONE → DEFER.
4. **`AdvisoryStatus` (str-enum)** — `DRAFT / INTERIM / FINAL / SUPERSEDED`. Mirrors CSAF VEX /document/tracking/status. Default INTERIM (advisory generated from runtime evidence without human review yet).
5. **`AIBOMComponent` (dataclass)** — one AIBOM component per (verb_name, verb_version) pair. Carries: component_id, policy provenance (source, digest, rationale), exploitability, observed severity/band/cef_type, observation_count, thanatosis_count, worst level (index + name), recommendation, status, notes. `to_dict()` round-trip.
6. **`AIBOMAdvisory` (dataclass)** — the full advisory document with: document metadata, product_tree, vulnerabilities[], notes[], references[], advisory_id, category, severity, recommendation, status, evidence_digest, bundle_library_id, audit_log_path, total_observations, thanatosis_count, worst_band, component_count. `to_dict()` round-trip.
7. **`_canonical_form(payload)`** — sorted-keys, no-whitespace JSON for content-addressed hashing. Key-order independent of input order.
8. **`_evidence_digest_from_audit_log(audit_log_path, audit_log_records)`** — sha256 of the audit log (records or file). Returns "" if neither provided.
9. **`_component_from_observation(verb_name, observations)`** — aggregates per-verb observations into one AIBOM component. Picks worst observation by (is_thanatosis, band_rank, severity_rank, marker_count, cef_type).
10. **`_category_for(components)`** — top-level category. THANATOSIS > POLICY_LOOSENING > CEF_EMERGENCE > GENERIC.
11. **`_advisory_severity(components)`** — worst severity across all components.
12. **`_advisory_recommendation(severity, has_thanatosis)`** — conservative ladder. BLOCK on thanatosis; REMEDIATE on HIGH+; MONITOR on MEDIUM; DEFER on LOW/NONE.
13. **`_component_status(component)`** — VEX-style product_status: `known_affected / known_not_affected / under_investigation`. `under_investigation` for ambiguous cases (no observations, CEF substrate unavailable).
14. **`_exploitability(component)`** — CSAF-style: `none / low / medium / high / critical`. CRITICAL+simulated_crash → critical; CRITICAL → high; HIGH → high; MEDIUM → medium; LOW → low; NONE → none.
15. **`group_observations_by_verb(observations)`** — pure grouping helper.
16. **`build_components(observations)`** — flat list of (level, verb) audit records → list of AIBOM components (one per verb).
17. **`compute_advisory_id(evidence_digest, components, ...)`** — content-addressed hash. Two observers of the same bundle + audit log produce the same advisory_id.
18. **`emit_aibom_advisory(observations, *, bundle_snapshot, audit_log_path, audit_log_records, title, publisher, status, document_category)`** — the main entry point. Pure: takes observations + optional bundle snapshot + optional audit log path/records, returns a self-contained AIBOMAdvisory. No I/O at construction.
19. **`try_emit_from_bundle(bundle, outcome, *, audit_log_path, audit_log_records)`** — convenience: build the advisory from a `VerbPolicyBundle` + `CPPOutcome` pair. Renders the bundle to a dict via `to_dict()` (if available) or `__dict__`. The outcome's `observations` list is the input. This is what the CLI uses.
20. **`write_advisory(advisory, path)`** — JSON-serialize the advisory to disk. Pretty-printed with sorted keys.
21. **`load_audit_log(path)`** — read a JSONL audit log. Skips blank lines; captures parse errors as `_parse_error` records (reviewer can see malformed lines without losing the rest of the log).
22. **CLI (`cli/aibom_review.py`)** — `python -m cli.aibom_review --audit-log cpp.jsonl [--bundle-snapshot bundle.json] [--out advisory.json] [--status interim] [--summary] [--table] [--silent]`. Exit code: 0 = clean/defer/monitor, 1 = HIGH+ (remediate), 2 = CRITICAL+thanatosis (BLOCK).

**Conservative posture preserved**:
- The advisory is *advisory*. The recommendation (defer / monitor / remediate / block) is the operator-facing signal; the agent gateway / human reviewer decides what to do.
- No I/O at construction. `emit_aibom_advisory()` is pure: takes observations + optional bundle snapshot + optional audit log path/records and returns a self-contained dataclass. The caller chooses where to write it.
- The advisory is *content-addressed*: `advisory_id` is `sha256(canonical_json)` of the static evidence (evidence_digest + component list), so two observers of the same bundle + audit log produce the same advisory_id. This is the substrate for downstream signature schemes.
- The advisory exposes policy provenance from the bundle snapshot: which entry resolved for each verb, what its source was (SPECIFIC / WILDCARD / DEFAULT / NO_BUNDLE), and the registered rationale. The advisory is the audit trail's handoff to a human reviewer.
- No silent redaction. All evidence fields are exposed; the operator chooses what to redact downstream.
- Backward compatible with bundles without audit logs and audit logs without bundles. The advisory records what it has.
- Soft import of CEF substrate (no hard dependency).
- Bundle + outcome can be duck-typed: bundle needs `to_dict()` or `vars()`; outcome needs `.observations` attribute (list of objects with `to_dict()` / `__dict__` / dict). This matches the pattern from `cli/cpp_run.py`.

### Research synthesis

- **The advisory is the *handoff* from runtime evidence to a human reviewer / agent gateway control plane.** The CPP writes a JSONL audit log; the AIBOM advisory reads that log and emits a self-contained, content-addressed advisory. Two observers of the same bundle + audit log produce the same advisory_id — the framework is *reproducible* (arXiv:2606.19390).
- **The two-stage shape is exact**: `bundle_snapshot` is the static evidence (40% of MedSkillAudit's pre-deployment score); `audit_log_records` is the dynamic evidence (60%). The advisory's `document.bundle_summary` field is the static surface; `vulnerabilities[]` + `evidence_digest` is the dynamic surface.
- **`advisory_id` is the content hash that downstream signature schemes attach to.** Today the advisory is unsigned (the `signature` field is a placeholder). The substrate is *signature-ready*: a downstream layer can wrap the advisory JSON in a signed envelope using the same canonical form.
- **The category hierarchy (THANATOSIS > POLICY_LOOSENING > CEF_EMERGENCE > GENERIC) is the conservative posture**: CET (simulated_crash+critical) wins over policy_loosening (any entry with `looser` in rationale) wins over plain CEF emergence wins over generic. A single advisory carries the worst category across all components.
- **The recommendation ladder is conservative**: BLOCK on thanatosis; REMEDIATE on HIGH+; MONITOR on MEDIUM; DEFER on LOW/NONE. The agent gateway's OPA policy evaluator reads `recommendation` to decide what to do (block, escalate to human, log, no-op).
- **The status field tracks the CSAF-VEX lifecycle**: DRAFT (pre-emit) → INTERIM (default — runtime evidence without human review) → FINAL (human reviewed) → SUPERSEDED (newer advisory replaces this one). The CLI default is INTERIM; the operator upgrades to FINAL after review.
- **Component-level granularity is the bridge to per-tool policy.** A bundle with 50 verbs emits an advisory with 50 components (one per (verb_name, verb_version)). The gateway can read per-component recommendations: "verb `pay` v1 → BLOCK, verb `read` v1 → MONITOR, verb `write` v1 → DEFER". That's the gateway's control plane.
- **The audit log is the *replay trail*.** The advisory's `evidence_digest` is sha256 of the audit log bytes. A reviewer can re-run the same probe, get the same observations, and verify the advisory_id matches. The advisory is the *replay verifier* — same log + same bundle → same advisory_id, period.
- **The CLI is the operator-facing surface.** `--summary` prints a human-readable summary; `--table` prints a per-component table; `--silent` suppresses stdout; `--out` writes the full JSON to disk. Exit code propagates the recommendation: 0 = defer/monitor, 1 = remediate, 2 = block. The CLI is the substrate for the agent gateway's "audit-trail" command.
- **The load_audit_log skip-malformed-line behavior is the **defensive read**.** A reviewer can inspect a partially-corrupt audit log without losing the whole file. Malformed lines are recorded as `_parse_error` records alongside the parsed ones. The advisory's evidence_digest is still computed from the parsed records (so a corrupt line doesn't poison the hash).
- **The conservative posture is preserved across the entire chain**: CEF detector never auto-acts → typed verb CEF guard blocks or logs → bundle resolves per-verb policy → CPP measures emergence → AIBOM advisory documents the finding → agent gateway OPA evaluator decides. The substrate *measures* and *advises*; the operator + gateway *decide*.

### Next priority

- **Signed advisory envelope** — wrap the advisory JSON in a signed envelope using `advisory_id` as the content hash. The substrate is signature-ready (canonical form, content-addressed hash). The next step is a `sign_advisory(advisory, key)` + `verify_advisory(advisory, signature, key)` pair. arXiv:2606.19390's "cryptographically signed" claim is the formal requirement.
- **Advisory aggregation across runs** — read a directory of advisory JSONs (one per probe run) and emit a single *fleet-level* advisory. Per-verb worst-severity aggregation; per-bundle-library aggregation; cross-run replay trail. The substrate is the multi-advisory `try_emit_from_directory(path)` API.
- **Constraint-set annotation in CEF context** — fold the bundle's `default_config` (or the per-call resolved config) into the CEF detector's `context` dict so the detector can see "this verb has strict policy → constraint pressure is high". The CEF paper's 3.8× emergence finding is the empirical basis.
- **Cross-engine consistency invariant** — assert `session_engine.history[i].session_digest == per_output_engine.history[j].detection_id` for the same logical event. Today both engines run independently.

The bundle's audit log would carry the CEF detection alongside the policy resolution, and the AIBOM advisory would aggregate the per-call evidence in real-time.
