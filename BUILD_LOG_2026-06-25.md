### Build Task: Session-level probabilistic trip bound (arXiv:2606.20510)

**Motivation**: the previous build (2026-06-24) added `ProbabilisticTripEngine` to the *per-output* path (`assess_cef_to_breaker`) but left the *session* path on a point estimate. A `CEFSessionVerdict` in `POINT_OF_NO_RETURN` was still boolean — the operator couldn't say "we expect ≤ X% of sessions to cross the recovery horizon in steady state." Today's build closes that gap by wiring the engine into `CEFSessionDetector`. The session verdict now carries the same `trip_probability` / `trip_upper_bound` / `trip_band` / `trip_n_samples` shape that the breaker outcome already exposes at the per-output layer.

### Key Components

1. **Soft import of `ProbabilisticTripEngine` / `TripBand` / `TripObservation`** in `core/cef_session.py`. Falls back to `None` when the probabilistic module is unavailable. The session detector still works exactly as before without an engine — no breaking change.
2. **`CEFSessionConfig.probabilistic_engine: Optional["ProbabilisticTripEngine"]`** — new optional field, defaulted to `None`. The frozen-dataclass invariant is preserved.
3. **`CEFSessionDetector._feed_probabilistic_engine(detection)`** — called once per `observe()` that *passed the severity floor*. Builds a `TripObservation(source="session", tripped=(state == POINT_OF_NO_RETURN), detection_id=..., session_digest=...)` and appends to the engine. The engine grows by exactly one entry per fabrication beat.
4. **Cached mirrors on the detector** (`_last_trip_probability`, `_last_trip_upper_bound`, `_last_trip_band`, `_last_trip_n_samples`): set by `_feed_probabilistic_engine`, consumed by `_build_verdict`. Avoids re-querying the engine on every build.
5. **`CEFSessionVerdict` gains 5 new fields** — all defaulted:
   - `trip_probability: float = 0.0`
   - `trip_upper_bound: float = 0.0` (sound DRO bound at `1 - alpha = 0.95`)
   - `trip_band: Optional[str] = None` (LOW / MEDIUM / HIGH / CRITICAL)
   - `trip_n_samples: int = 0`
   - `has_probabilistic_engine: bool = False`
   All five appear in `to_dict()`. The 25 pre-existing tests (which do not configure an engine) all still pass with `has_probabilistic_engine=False`.
6. **`CEFSessionVerdict.trip_summary()`** — returns `None` when no engine; otherwise a dict mirroring the engine's state for audit (point estimate, bound, band, sample count).
7. **`reset()` clears session mirrors but does NOT clear engine history.** The engine is multi-session by design; the session is a window into it. Mirrors reset so subsequent `observe()` calls start clean within the new session.

### Conservative posture (paper invariants preserved)

- **Opt-in**: no engine = same behavior as before. `has_probabilistic_engine=False` is the safe default.
- **Additive**: the boolean `state` + `action` + `is_compromised()` are unchanged. The bound is a measurement on top, not a replacement.
- **Read-only on the engine**: the detector feeds via `update()` but never mutates the engine on read.
- **Below-floor observations are not fed.** They are "clean beats" that reset the consecutive run; feeding them would inflate `n_samples` with trivially-no-trip data and dilute the bound's tightness. The conservative posture is "the engine measures *real* trips, not the absence of all observation."
- **`detect_cef_session(...)` is unchanged** — the end-to-end convenience still works exactly as before; no caller needs to change.

### Research Synthesis

- The 2026-06-24 build made the *per-output* path measurable. Today's build makes the *session* path measurable. Together they form a two-layer safety substrate where the operator sees both layers' bands in one place.
- The 2026 consensus ("stateless LLM + stateful runtime", genalphai.com; Springer agentic AI review) is that the stateful runtime is where the *measurable* safety lives. Our session engine is the safety analog: persistent engine state across turns, with bounds that make the state verifiable.
- "Hippocampal Explicit Memory" (arXiv:2606.11245v1) argues explicit memory is the cornerstone for higher-order AGI capabilities. The engine's history *is* the agent's explicit safety memory across sessions — and the bound is the operationally-useful projection of that memory.
- The Hephaestus (v0.7.23, Jun 25), Hermes Agent v0.15.0, Continuum, and OpenLAIR/OpenSkill projects are all converging on the same architectural pattern: a thin verifier layer above a stateful memory layer. `ProbabilisticTripEngine` + `CEFSessionDetector` is the closest functional equivalent in our substrate, with the bound as the verification gate.

### What I learned building this

- The per-output `assess_cef_to_breaker` was already wired because `assess` returns a `CEFBreakerOutcome` and the engine plugs in via `CEFIntegrationConfig`. The session path needed a separate integration because the session detector returns a *different shape* (`CEFSessionVerdict`) and the engine semantics for *sessions* differ from per-output: a session "trips" only at `POINT_OF_NO_RETURN`, not at every fabrication.
- Mirroring that semantic exactly keeps the bound meaningful — "upper bound on fraction of sessions that cross the recovery horizon" is a steady-state safety claim an operator can act on, not a per-beat noise signal.
- The soft-import pattern (try-except at module load, fall back to `None`) means the module works in slim installs that drop scipy, while still benefiting from the engine when it's available. This matches the conservative posture of every other CEF module.
- `detect_cef_session()` (the end-to-end convenience) doesn't take an engine today. Adding `probabilistic_engine` to *its* signature would be a backward-compatible change but is Q3 work; for now, callers that want the engine wire up `CEFSessionDetector` directly.

### Next Priority

- **Governor circuit gating**: a `GovernorCircuit` that opens when `trip_upper_bound >= trip_threshold` and only re-closes when the bound drops below a reset threshold. The bound becomes the *policy input*, not just a measurement. Natural next layer on top of today's wiring.
- **`assess_cef_to_breaker` session-band integration**: when a session is in `POINT_OF_NO_RETURN`, surface the *session* band alongside the *per-output* band in the breaker outcome. The operator sees both layers' measurements in one place.
- **Multi-session engine persistence**: today the engine is in-memory. For fleet-scale, the engine should be backed by an append-only JSONL so the bound survives restarts.
