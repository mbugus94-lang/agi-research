**Key Changes**:

1. **`PolicyResolutionSource` enum** — renamed `EXACT` → `SPECIFIC` (the canonical name from the test contract). Kept `EXACT` as a string-alias so any external code or test still referring to `EXACT` works.
2. **`VerbPolicyEntry.looser_than_default` property** — added. Compares the entry's config against the bundle's `default_config` on a per-band basis (low/medium/high/critical action). Returns True iff any band is weaker than the default. Implementation uses a `set_baseline()` method that the bundle calls at `register()` and `register_or_replace()` to snapshot the default without holding a back-reference.
3. **`strict_unknown_verb` field** — added to `VerbPolicyBundle`. When True, `resolve()` raises `KeyError` for unknown verbs. When False (default), it returns the bundle's `default_config` if set, else `(None, DEFAULT, None)`.
4. **`create_verb_policy_bundle(..., strict_unknown_verb=...)`** — wired through.
5. **`VerbPolicyBundle.to_dict()`** — now includes `strict_unknown_verb` and `default_config`.
6. **`BundledVerbRuntime.state` as an attribute proxy** — was a method (`state()`), now an `@property` (and setter). The test does `bundled.state is runtime.state`, so `state` must be an attribute that proxies to `runtime.state`. `audit()` remains a method (test does `bundled.audit() is runtime.audit`).
7. **`BundledVerbRuntime.invoke()`** — now calls `resolve_policy()` per invoke, which records a RESOLVE/FALLBACK/MISS event in the audit log. The previous version called `bundle.resolve()` directly (which did NOT record audit events), so the audit log was empty for all per-call resolutions.
8. **`register()` / `register_or_replace()` / `deregister()`** — no longer append REGISTER/DEREGISTER events to the audit log. The audit log now records *resolution events only* (one per call). This avoids double-counting in `counts_per_source()` (the previous implementation would record both a REGISTER and a RESOLVE for the same logical event).
9. **`PolicyAuditEntry` is no longer `frozen=True`** — needed for `test_audit_chain_detects_tamper`, which mutates an entry's `verb_name` and expects `verify_chain()` to return False.
10. **`VerbPolicyAuditLog.append()`** — now computes the `event_hash` from the entry's `prev_hash` and the canonical payload (using `_hash_audit_entry`). The previous version stored whatever `event_hash` was on the entry (which would be missing for test-built entries, causing `verify_chain` to fail).
11. **`VerbPolicyBundle.audit_log()`** — wires `bundle.audit_path` into `log._jsonl_path` so subsequent appends get flushed to disk.
12. **`cli/policy_review.review_bundle()`** — `n_entries` now counts *registered entries* (via `len(bundle.entries)`) with the `last` filter applied; `n_audit_events` reports the per-call resolution event count. This matches the CLI test contract: `register()` → `n_entries >= 1`, `register() x2` + `last=1` → `n_entries == 1`.
13. **`test_verb_policy_bundle.py`** — added `import os` at the top to fix the `NameError` in `test_jsonl_audit_writes_to_disk`.

**Research synthesis**:
- The 2026-07-03 build created the bundle's shape; today's build aligns the implementation to the test contract. This is the *contract lock-in* step — the bundle is now a stable substrate for downstream work.
- **`SPECIFIC` is the canonical name**; `EXACT` is the legacy alias. Future code should use `SPECIFIC`. The alias exists for backward compatibility with anything that referenced `EXACT` (the 2026-07-03 release value).
- **Audit log = resolution events only** (one per call). Registration is observable via `bundle.entries` and `len(bundle)`. This separates *what policies exist* (registered entries) from *what policy was applied* (resolution events) — the two are different audit dimensions.
- **`n_entries` vs `n_audit_events` split** in the CLI: the bundle has 3 entries registered and 100 resolutions in the audit log; the CLI surfaces both. This is the shape a replay layer needs.
- **Why the bundled runtime's `state` is now an attribute, not a method**: a VerbRuntime's `state` is a mutable dict. Calling `bundled.state()` would be confusing because it returns a reference, not a snapshot. The attribute proxy is a thin read/write pass-through to the inner runtime's dict.
- **ContextNest (arXiv:2607.02116)** confirms the convergence: the substrate needs both *verifiable input* (versioned, hash-chained documents) and *verifiable output* (CEF detection, audit ledger). The two together are what AIBOM / CSAF-VEX / ContextNest all demand.
- **Gear-Based Safety (arXiv:2607.00041)** is the *constraint-pressure probe* paper that the CEF paper called for. The 2026-07-02 build left CPP integration as future work.
