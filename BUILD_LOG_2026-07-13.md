### Why this is the right build for today

The substrate has *two* compositional gates (Prismata read side, DSCC write side). Both are exposed as Python modules. Neither was exposed as a CLI. The next-priority from 2026-07-12 was: "wire the Prismata read-side into an operator CLI". I built that CLI today and added a JSON-driven combined mode that uses `dual_gate_check` as the single API surface.

This unblocks two follow-on builds:
- **CAGE-1 evaluation report CLI** — `python -m cli.cage1_report` can now read the loop's `reports` list and emit a CAGE-1-shaped evaluation with the new DSCC + CLP columns.
- **External agent harness integration** — Omnigent, eve, ag-kit, etc. can now call `python -m cli.clp_check` as a "policy tool" to check chains before they run, using JSON in / JSON out.

### Substrate invariant today

> Every chain verdict in the substrate is now CLI-readable.
> Read side: `python -m cli.clp_check --capabilities ... --chain ...`
> Write side: `python -m cli.dscc_clearance --policies ... --chain ...`
> Combined: `python -m cli.clp_check --capabilities ... --chain ... --write-policies ...`
