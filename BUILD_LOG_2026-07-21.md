# Build Log — 2026-07-21: CAGE-1 Fleet Envelope in Report CLI

## Research

Reviewed AGI and agent-architecture work from the preceding two weeks, with emphasis on long-horizon execution, memory, verification, and self-improvement boundaries.

- **StructAgent** (arXiv:2607.11388, July 2026) uses a compact causal task state with verifier-backed transitions. The reported OSWorld-Verified gains reinforce the project’s decision to preserve evidence, lineage, and explicit unmeasured dimensions instead of relying on final scores alone.
- **TopoAgent** (arXiv:2607.14658) replaces linear multimodal reasoning with a state-isolated dependency graph that supports self-correction. This is a useful architectural signal for future fleet reports: preserve dependency and session order rather than flattening all runs into totals.
- **DeepSearch-World** (arXiv:2607.07820) uses deterministic web tools, progress verification, grounded reflection, and failure recovery for self-distillation. It supports replayable fixture inputs and fail-closed parsing as evaluation infrastructure.
- **Reward-Free Evolving Agents via Pairwise Validator** (arXiv:2607.14408) uses parent/child comparison in place of a scalar reward. It reinforces review-only improvement: comparison can surface evidence, but this repository must not auto-apply mutations.
- **Speculate with Memory** (arXiv:2607.12236) combines transition, episodic, and confusion memories for lossless acceleration. The important boundary is separation of observational analysis from the trajectory being measured.
- **DeepStress** (arXiv:2607.13920) stress-tests search agents under controlled evidence quality. This supports retaining missing or malformed retrieval evidence as explicitly unmeasured/anomalous rather than coercing it into a success or regression.
- **ABot-AgentOS** (arXiv:2607.10350) combines multimodal graph memory, context-isolated skills, verification, and gated failure-driven evolution. Its provenance and promotion gates map closely to the project’s read-only, evidence-bounded posture.

Open-source signals were also checked. **HKUDS/OpenSpace** emphasizes a quality-first skill hub with runtime state, tool grounding, security, and a versioned skill-evolution system; **Tracer-Cloud/OpenSRE** combines AI SRE agents with an open reinforcement-learning environment and synthetic incident simulations; **HKUDS/OpenPhone** pairs an on-device agent model with a pre-built navigation graph and VLM fallback. These are architecture/activity signals from public repository pages, not a controlled popularity ranking.

## Build: one focused task

Implemented task D/C: a read-only integration that exposes the existing CAGE-1 fleet aggregator through the broader report CLI.

- `cli/cage1_report.py` now accepts `--fleet-input PATH` for ordered JSON-array or JSONL fleet snapshots.
- Fleet output supports the existing `--format {markdown,json,both}` contract.
- The report CLI preserves fleet session order, digest lineage, anomalies, per-dimension summaries, and explicit unmeasured evidence.
- `experiments/test_cage1_report_cli.py` adds JSONL fleet-mode coverage and verifies that the input file is not mutated.
- The CLI is source-exclusive through argparse: fleet input, snapshot comparison, audit log, or demo are separate modes.

**Safety boundary:** this change only reads, aggregates, and renders. It does not change policy, repair memory, alter retrieval, deduplicate sessions, or apply self-improvement. Invalid evidence remains visible as invalid or unmeasured.
