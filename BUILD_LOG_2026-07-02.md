## Phase 3: Build — Typed Verb CEF Guard (closes 2026-07-01 carryover)

**Build Task**: Wire the existing CEF/CET detector (`core/cef_detector.py`, arXiv:2606.14831) into the typed-verb runtime (`core/typed_verb_library.py`, OpenReview PxbzZPlPhr) so that any string-valued output a verb returns gets scanned for constraint-evasive fabrication patterns, and the operator's policy maps severity → action (NONE/LOG/FLAG/HALT/ESCALATE).

**Motivation**: The 2026-07-01 build landed the typed-verb layer as a deterministic substrate for structured web actions — but the *handlers* are pluggable (LLM calls, browser automation, tool invocations). A handler can return a string that fabricates an external obstacle ("Blocked by audit and regulatory compliance..."), and the typed-verb runtime would pass it through to the next program step or to the user. The CEF detector is the output-inspection layer; today's bridge makes the two substrates cooperate.

**Components**:
1. **Soft CEF import** — `core/typed_verb_cef_guard.py` imports the CEF detector lazily; missing substrate → no-op guard with `CEFGuardAvailability.NO_CEF_SUBSTRATE`. The typed-verb runtime never breaks because of a missing CEF layer.
2. **String harvesting** — `_collect_strings` walks the verb's output mapping, picks string-valued fields and JSON-renders dict/list values, truncates each to `max_string_length_per_field` (default 8192), caps total scanned fields at `max_fields_to_scan` (default 32), sorts by key for determinism.
3. **Stable aggregate hash** — `_aggregate_text_hash` returns sha256 of `key=value` joined pairs, sorted by key. Re-ordering the same output dict produces the same hash.
4. **`VerbCEFGuard` (pure)** — takes a `CEFDetector` + `VerbCEFGuardConfig`, exposes `inspect_output(...)`. Never mutates, never raises, never short-circuits.
5. **`VerbCEFGuardConfig`** — operator-controlled severity → action mapping. Defaults: LOW→LOG, MEDIUM→FLAG, HIGH→HALT, CRITICAL→ESCALATE.
6. **`VerbOutputCEFInspection`** — frozen dataclass with scanned_fields, aggregate_text_hash, detection, guard_action, is_clean, availability. `to_dict()` round-trip.
7. **`GuardedVerbRuntime`** — wraps a `VerbRuntime` + `VerbCEFGuard`. `invoke()` runs the inner runtime, inspects output, appends to `inspections`, and (if `halt_on_halt=True` AND guard_action ∈ {HALT, ESCALATE}) returns a new `VerbStepResult` with `success=False` and `error="CEF guard {action}: {rationale}"`.
8. **`halt_on_halt=False` mode** — observe-without-blocking for dashboards and gradual rollout.
9. **`record_cef_to_evidence_ledger(inspection, ledger)`** — writes CEF detections to `EvidenceLedger` with `source="typed_verb_cef_guard"`, call_id, verb metadata. Bridges to the existing IEEC / PCA bridge substrate.

**Conservative posture**:
- Soft CEF import (slim-install safe)
- Guard never auto-escalates; only reports
- Operator controls every severity band
- `min_output_length=12` floor
- `max_fields_to_scan=32` and `max_string_length_per_field=8192` caps
- Hash sorted by key for determinism
- Wrapper, not modification — existing verb programs unchanged
