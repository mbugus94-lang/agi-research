### Motivation

The June 21 build (`CEF Substrate Integration`, 33 tests) introduced the
*boolean* bridge: a CRITICAL per-output detection trips the breaker; a
session-level `POINT_OF_NO_RETURN` trip does too. Between the two sits a
fundamental safety-utility gap: an agent that fabricates 1% of outputs is
qualitatively different from one that fabricates 30%, but the boolean
treats both as "below floor -> no trip" or "above floor -> trip" with no
intermediate state.

arXiv:2606.20510 (Jun 20 2026, "Efficient and Sound Probabilistic
Verification for AI Agents") closes the gap with a *distributionally-robust*
(DRO) Bernoulli upper bound on the policy-violation probability. The bound
is **sound** (true rate Ōēż bound with probability Ōēź 1 - alpha) and
**distribution-free** (no independence / sub-Gaussianity assumption). The
paper's empirical claim: the bound is *tight* ŌĆö not conservative-for-no-
reason ŌĆö and non-trivial even with a few dozen observations.

The June 22 build (`AgentLoop Ōåö CEF Substrate Bridge`) closed the loop-
level integration. Today's build closes the *measurement layer*: every
trip/no-trip event is fed through a DRO engine, and the outcome carries a
sound upper bound + a discrete action band.

### Key Components

1. **`BetaPosterior`** ŌĆö conjugate Beta prior/posterior over a Bernoulli
   trip rate. `mean`, `variance`, `upper_bound(confidence)` (Bayesian
   quantile bound). Default prior `Beta(1, 1)` is uninformative; operators
   can pass `alpha_prior` / `beta_prior` for a tighter baseline.

2. **`dro_bernoulli_upper_bound(n_success, n_total, confidence)`** ŌĆö the
   paper's DRO bound = `min(Hoeffding, Clopper-Pearson exact)`. Tighter
   than Hoeffding at small n or extreme rates; dominates for moderate n.
   Monotone in `n_total` and `confidence`. `n_total == 0` returns 1.0
   (no information -> worst case).

3. **`TripBand`** ŌĆö discrete bands (`LOW` < 0.01, `MEDIUM` < 0.05,
   `HIGH` < 0.20, `CRITICAL` Ōēź 0.20). `band_for_upper_bound(bound,
   thresholds)` is monotone in the bound. Default thresholds are
   conservative.

4. **`TripObservation`** ŌĆö frozen dataclass for one trip / no-trip event
   (`tripped`, `source` ("per_output" / "session"), `detection_id`,
   `session_digest`, `timestamp`). `to_dict()` for audit round-trip.

5. **`ProbabilisticTripEngine`** ŌĆö composes posterior + DRO bound + history.
   `update(obs)` is the only mutator and returns `self` for chaining.
   `trip_probability` (posterior mean), `trip_upper_bound` (DRO bound),
   `trip_band` (discrete), `bayesian_upper_bound` (for sanity check),
   `summary()` / `to_dict()` for audit. `reset()` clears the history;
   the prior is preserved.

6. **Integration into `assess_cef_to_breaker`**: when
   `CEFIntegrationConfig.probabilistic_engine` is set, every assessment
   feeds the engine and the `CEFBreakerOutcome` carries
   `trip_probability` / `trip_upper_bound` / `trip_band` /
   `trip_n_samples`. Default = engine is opt-in; existing call sites are
   unaffected.

7. **`breaker` made optional in `assess_cef_to_breaker`**: the function
   is pure (no mutation); callers can run it standalone to drive the
   probabilistic engine without touching a breaker. Existing call sites
   that pass a breaker still work.

### Conservative Posture

- Default `confidence = 0.95` (sound at 95% level).
- Default prior `Beta(1, 1)` (uninformative, max-entropy).
- Empty history -> bound = 1.0 (no info -> worst case). Band in that
  state is `CRITICAL` ŌĆö the operator sees "we have no signal" as the
  loudest possible state.
- Bands monotone in the bound (a higher bound yields a higher band).
- The engine never demotes historical records; `reset()` is explicit.
- The boolean `verdict` + `should_open` are unchanged ŌĆö the engine is
  *additive*, never replacing the breaker.
- `n_total == 0` is treated as the absence of evidence, not as
  evidence-of-absence. The bound's worst-case (1.0) is the loud
  "look here first" signal.

### Research Synthesis

- arXiv:2606.20510 (Jun 20 2026) is the *measurement* layer. The
  `CEFDetector` + `CEFSessionDetector` produce a point estimate; the
  `SafetyCircuitBreaker` is the *policy* layer (boolean). The DRO
  engine sits in between: it *measures*, and the bound becomes a
  measurable safety claim, not an ad-hoc per-incident verdict.
- The DRO bound's "min(Hoeffding, Clopper-Pearson)" pattern is the
  paper's "best of both worlds": Chernoff dominates at moderate n;
  exact dominates at small n or extreme rates. We mirror this with
  the same `min` selection.
- The `TripBand` is the conservative analog of the CEF severity ladder
  (`LOW` / `MEDIUM` / `HIGH` / `CRITICAL`). Bands make the bound
  *operationally* useful: the operator can map bands to actions
  (log, warn, freeze, halt) without re-reading the bound every time.
- The `Beta(1, 1)` prior is the *uninformative* choice. Operators with
  a baseline trip rate can pass `alpha_prior` / `beta_prior` for a
  tighter prior ŌĆö but the conservative default is "we don't know, so
  the bound is loud".
- The `n_total == 0 -> CRITICAL` rule is the conservative analog of
  `classify_risk` promoting any proposal that touches the self
  surface: absence of evidence is *not* evidence of absence.
