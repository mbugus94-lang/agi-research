### Build Task: Memory Probe (MEMPROBE-style Recoverability Harness)

**Motivation**

The MEMPROBE paper introduced the first benchmark that *directly* measures whether an LLM agent's memory preserves a structured hidden user-state. Their headline finding: strong **task performance** is a poor proxy for **memory fidelity** — recovery was ~0.6 even on systems with high task-completion, and top-k retrieval degraded it further. Our repo already has a
rich governance + memory substrate (EvidenceLedger, SafetyCircuitBreaker, **TieredMemorySystem**, CEF session detector, governor circuit). Today's build is the **measurement layer** for that substrate — a deterministic harness that exposes memory fidelity vs task-completion on the *same* hidden user, the same way the Governor Circuit exposes ring-routing vs task-completion.

**Headline result (today's run):**

| Target            | recovery | task_completion | fidelity_gap | topk_degradation |
|-------------------|---------:|----------------:|-------------:|-----------------:|
| in_memory (gold)  | 1.00     | 0.40            | -0.60        | +0.33            |
| tiered_memory     | 1.00     | 0.38            | -0.62        | **+0.00**        |

Both targets fully preserve the hidden user-state (recovery=1.00), but task-completion is far below 100% because the probe asks **paraphrased questions** and uses **top-k snippet retrieval** — the same surface that MEMPROBE found degrades retrieval-augmented agents. The **fidelity_gap = -0.60** is the diagnostic: the agent appears to remember while quietly losi
ng exact retrieval fidelity.

**What was built**
- `core/memprobe.py` — new benchmark module with:
  - `HiddenFact`, `UserState`, `Scenario`
  - `MemoryProbeTarget` protocol
  - `InMemoryTarget` + `TieredMemoryTarget` adapters
  - `ProbeResult` with recovery/task/fidelity metrics
  - `run_probe`, `compare_targets`, `summarize`, `quick_probe`
  - default 12-dimension hidden state + 50-scenario suite
- `experiments/test_memprobe.py` — 36 new tests covering matcher semantics, scenario scoring, protocol adapters, run_probe, summary, conservative posture, and headline-gap behavior
- `core/__init__.py` — exported the new MEMPROBE symbols

**Verification**

- `experiments/test_memprobe.py + experiments/test_tiered_memory.py + experiments/test_recursive_self_improvement.py + experiments/test_governor_circuit.py`: 147/147 pass

**Why this matters**
- The paper's claim becomes an internal metric: memory quality must be measured directly, not inferred from task completion
- The harness makes paraphrase loss and top-k degradation visible
- Tiered memory's `topk_degradation=0.00` on the benchmark is a concrete before/after signal for future memory refactors
- The probe is intentionally deterministic so future runs can compare apples to apples
