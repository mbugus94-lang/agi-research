# Build Log — 2026-07-24 — Verification-Only CAGE-1 Decision Consumer

## Research synthesis

The last two weeks of agent research continued to converge on evidence-bounded autonomy:

- **Knowledge-Centric Self-Improvement** (arXiv:2607.19592) keeps the agent disposable and improves a curated, evidence-grounded knowledge base instead. This supports preserving source evidence and refusing silent repository mutation.
- **Programmatic Memory Enables Long-Horizon Reasoning** (arXiv:2607.20064) reports that frontier models without harnesses perform poorly on interactive, long-horizon environments; explicit programmatic memory and executable state are the useful substrate.
- **Evaluating Scientific Memory as Budgeted Context** (arXiv:2607.16848) treats memory as a budgeted, evaluated resource rather than an opaque store, supporting explicit coverage and provenance fields.
- **AI Agents Do Not Fail Alone: The Context Fails First** (arXiv:2607.14275) makes context quality an auditable reliability layer across instructions, tools, grounding, injection hardening, and token efficiency.
- **MRMS** (arXiv:2607.04617) synchronizes structured, vector, and graph memory before context projection; the relevant implementation lesson is to verify that evidence layers still refer to the same content before consumption.

Open-source activity signals included **agentscope-ai/AgentTeams** (manager-workers, multi-runtime collaboration, and human-in-the-loop by default), **NousResearch/hermes-agent v0.19.0** (durable delivery ledger and smart approvals), **aget-framework/aget** (persistent shared domain intelligence with lifecycle governance), and **triggerdotdev/trigger.dev** (durable tasks, retries, queues, and observability). These are architecture/activity signals, not controlled popularity rankings.

## Build: one focused task

Implemented the next priority from the signed-decision build: a verification-only consumer/report path.

- Added `core/cage1_decision_consumer.py` with `consume_operator_decision(...)` and a serializable `DecisionConsumerReport`.
- Joins a CAGE-1 advisory, one or more signed decisions, and the preserved raw trend/fleet envelope.
- Verifies signatures, advisory content digests, raw-evidence equality, missing/invalid/ambiguous/conflicting decision states, and returns an explicit status.
- Added `cli/cage1_consume.py` for JSON input, repeated signed envelopes, HMAC key verification, optional output, and non-zero status for invalid or non-consumable reports.
- Exported the consumer API from `core/__init__.py`.
- Added five tests covering valid joins, missing evidence, tampered evidence, conflicting valid decisions, and CLI output.

## Safety boundary

The consumer is strictly verification-only. `decision_applied` and `automatic_action_taken` are always false. It does not apply `accept`, `reject`, or `defer`; it does not mutate policy, source evidence, repository files, or self-improvement state. Conflicting and invalid records remain visible for operator review.
