## Build Action 1: `cli/governor_circuit.py` (carry-over priority)

**Motivation**: every other substrate module exposes a CLI (governed_action_loop, silent_failure_monitor, evidence_ledger...). The 2026-06-29 build documented the carry-over priority to give GovernorCircuit the same surface. Closed today.

**What it provides** (4 subcommands):

1. **`summary --state-file <path>`** -- print the gate's current state, bound, band, transition count, and per-layer summaries from a JSON snapshot. Idempotent; never mutates input.
2. **`audit-tail --path <JSONL> [-n N]`** -- tail the last N transition records from the audit JSONL. Skips malformed lines with a structured warning.
3. **`simulate --history <JSONL>`** -- drive a fresh `GovernorCircuit` from a JSONL history of `{tripped, label, layer?}`, applying defaults that mirror the substrate's CLI conventions. Prints the final state, bound, band, and full transition log so the operator can see what *would* happen.
4. **`doctor [--state-file PATH] [--audit PATH]`** -- quick health check: bound, band, state, transition count, last transition timestamp, health verdict (`OK`/`WARN`/`CRITICAL`).

**Design choices**:
- All subcommands emit pure JSON (or an `{"error": "..."}` envelope) so they compose with `jq`, dashboards, or other agents. No ANSI noise, no progress spinners.
- Simulation is **side-effect-free** (creates a fresh gate, never reads production state). The `doctor` subcommand never mutates either.
- Audit-tail is **safe** by default: it's the only subcommand that reads the full audit path, and it tolerates malformed lines (a corrupted JSONL does not crash the run).
- All subcommands honour `--help` with one short line of context.

**CLI smoke** (operator-facing, all commands mirror real substrate usage):
```
$ python3 cli/governor_circuit.py summary --state-file state.json
{ "state": "closed", "upper_bound": 0.0, ... }

$ python3 cli/governor_circuit.py audit-tail --path audit.jsonl -n 3
{ "path": "...", "lines_returned": 3, "transitions": [...] }

$ python3 cli/governor_circuit.py simulate --history history.jsonl
{ "final_state": "closed", "transitions_added": 7, ... }

$ python3 cli/governor_circuit.py doctor --audit-path audit.jsonl
{ "verdict": "OK", "state": "closed", "bound": 0.0, ... }
```

## Build Action 2: `cli/calibrate.py` (carry-over priority)

**Motivation**: 2026-06-25 build call-out -- "a `cli/calibrate.py` that takes a baseline trip rate and prints the recommended `alpha_prior` / `beta_prior` for the conservative-vs-tightness tradeoff." Closed today.

**What it provides** (3 subcommands):

1. **`recommend --target <float> [--pseudo-observations N]`** -- compute the Beta-prior pair `(alpha_prior, beta_prior)` whose posterior mean is the operator's target trip rate. Default pseudo_observations=20 (conservative); reduce for more reactive priors, raise for tighter bounds.
2. **`recommend --target <float> --sweep "5,10,20,50,100"`** -- same as above but over a list of pseudo-observation counts, returning all prior pairs in one JSON. Lets an operator pick the prior that matches their conservatism preference.
3. **`verify --target <float> --pseudo-observations N --trips T --total K`** -- take a recommended prior, feed `T` trips and `K-T` ok observations into a fresh `ProbabilisticTripEngine`, and print the resulting bound band, empirical rate, and engine summary. **Round-trips** the math: the same model the gate consumes is what `verify` exercises.

**Design choices**:
- Posterior-mean-only Beta prior: for target `t` and pseudo-observations `N`, set `alpha = t*N`, `beta = (1-t)*N`. Closes to `alpha=1, beta=N-1` so the prior is never degenerate at small `N`.
- Hard errors on `--target 0` or `--target 1` (a uniform Beta prior with zero pseudo-observations is non-informative; a degenerate prior isn't one at all).
- `verify` reuses the production `ProbabilisticTripEngine` math; a "verified" prior is one that has survived a synthetic workload.
- Output is always JSON; all errors return `{"error": "..."}` envelopes.

**Test coverage**: 20 tests, all pass:
- `TestRecommendPrior` (4): default weight 20, custom weight, target=0/1 errors, negative weight error, posterior mean matches target

## Build Action 3: `core/teachability.py` (research-driven, NEW)

**Motivation**: the most-cited 2026-06 arXiv paper this week is the ICML 2026 position paper "Agentic Safety is an Epistemic Property, Not a Behavioral One" (OpenReview 30mapdhNKH). It argues safety for advanced AIs is a **longitudinal property** of the evolving learner, not a behavioural snapshot of the current policy. The paper proposes **teachability** as the operational metric: the ability to preserve future corrective leverage under bounded intervention.

Our substrate already has the ingredients (stateful gate, audit ledger, recovery semantics). Today translates **teachability** into a runnable score for the existing `GovernorCircuit` so the operator can ask "*is my substrate still teachable?*" and get a number.

**Design**:

The score is a **weighted geometric mean** of 4 orthogonal sub-scores, each in `[0, 1]`:

1. **Recovery responsiveness** -- does the gate recover from OPEN when observations turn good? A "fully-taught" substrate has at least one recovery per ~5 elevated transitions.
2. **Intervention friction** -- given an audit trail of operator interventions, how *frictionless* is correction? (high friction = lots of "intervention failed, intervention retried" stamps).
3. **Audit completeness** -- does each transition record carry timestamp, reason, layer id, and **prior state**? An audit-trail-free gate is unteachable by definition (the operator can't see what to correct).
4. **Chatter resistance** -- does the gate flap around the trip line, or stay put? A flapping gate erodes corrective leverage (every intervention can be undone by the next observation).

The geometric mean penalises low outliers (one bad sub-score drags the whole teachability down), matching the paper's claim that **longitudinal** safety is a *conservative* property.

**Bands**:
- `teachability >= 0.66` -- HIGH (well-taught)
- `0.33 <= teachability < 0.66` -- MEDIUM (still teachable, attention warranted)
- `teachability < 0.33` -- LOW (corrective leverage at risk)

**Public surface**:
- `recovery_responsiveness(...)`, `intervention_friction(...)`, `audit_trail_completeness(...)`, `chatter_resistance(...)` -- pure helpers, each testable in isolation
- `TeachabilityScore` -- frozen dataclass with sub-scores, totals, intervention/recovery/transition counts, and human-readable notes
- `compute_teachability(*, transitions, audit_records, ...)` -- top-level scorer
- `teachability_from_circuit(circuit, *, audit_records=...)` -- convenience that reads `circuit.to_dict()` and computes the score

**Bug found and fixed during the build**: `compute_teachability` was hard-coded to look for `"OPEN"` and `"HALF_OPEN"` (uppercase) when counting recovery events, but `GateState` enum values are lowercase (`"open"`, `"half_open"`). On a real circuit that hit a trip+recover cycle, the count returned 0 and `teachability` came out as 0.46 instead of ~0.76. Fixed with `.lower()` normalisation that works for both fresh `GateTransition` dataclasses and JSONL-imported dicts. Added a regression test that verifies both shapes.

## Research Synthesis

**The 2026-06 trend**: the ICML position paper "Agentic Safety is an Epistemic Property, Not a Behavioral One" + arXiv:2606.24370 "Helpfulness Overrides Causal Caution" + A-MemGuard + Unbiased Principles Robust Rewards all converge on the same **shift from behavioural to epistemic safety**:

| Old (behavioural) | New (epistemic) |
|---|---|
| Does the model act safely on input X? | Does the model *preserve* the ability to be made safe on future X? |
| Per-output audit trail | Longitudinal ledger + intervention friction metrics |
| Threshold-tuned | Score-distributed (geometric mean over sub-scores) |
| "It's safe" | "It's teachable" |

Our substrate already does the **measurement** half of this (ProbabilisticTripEngine gives a sound bound; GovernorCircuit applies policy). Today's teachability score closes the **observability** half: an operator can now ask "*is my substrate still teachable?*" with a single number, and the score rewards the substrate behaviours (recovery, audit completeness, low chatter) that preserve corrective leverage.

The Causal Caution paper points at a complementary direction: **context-dependent suppression** of cautious reasoning -- the same kind of latent failure mode that audit-trail-completeness and intervention-friction subscores would catch (an audit record is the *fingerprint* of suppressed caution). Both are layers of the same longitudinal substrate.
